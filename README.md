Six progressively harder digital systems projects, written in Verilog and synthesized on real Xilinx FPGA hardware. This isn't simulation-only work — every module was placed, routed, and tested on the board.

The arc: **auto-generated HDL → combinational logic → serial comms → FSM control → image processing pipeline → USB host interface.**

## Projects

### ⚡ Lab 1 — Fixed-Point Audio Filter via HDL Coder
> Simulink model → synthesizable Verilog, automatically

Designed a **3-tap FIR filter** in Simulink using `sfix16_En14` fixed-point arithmetic to strip a 50Hz hum from a `.wav` audio file. Used **MATLAB HDL Coder** to auto-generate a fully synthesizable, clock-enabled Verilog module — then validated the output waveform against the raw input.

```verilog
// Auto-generated by HDL Coder — production-ready RTL from a Simulink block diagram
module HDL_DUT (clk, reset, clk_enable, in1, ce_out, out);
  input  signed [15:0] in1;  // sfix16_En14
  output signed [17:0] out;  // sfix18_En14
  // 3-stage delay line + fixed-point multiply-accumulate
  assign Gain_mul_temp = 16'sb0111100110111010 * Delay1_out1;
```

`MATLAB/Simulink` `HDL Coder` `Fixed-Point` `FIR Filter` `Audio DSP`

### 🖥️ Lab 2 — Binary-to-7-Segment Decoder
> Full synthesis, place-and-route, and timing closure on Spartan hardware

A synchronous **4-bit hex decoder** driving a physical 7-segment display. All 16 input combinations mapped and verified in behavioral simulation before hitting the board. Includes timing analysis reports, PAR output, and ISIM waveforms.

```verilog
module binaryto7seg(input clk, input [3:0] input_4bit, output reg [6:0] output_7seg);
  always @(posedge clk) begin
    if      (input_4bit == 4'b0000) output_7seg = 7'b0111111; // 0
    else if (input_4bit == 4'b1000) output_7seg = 7'b1111111; // 8
    // ... all 16 cases mapped
  end
```

`Combinational Logic` `Xilinx ISE` `Timing Analysis` `Physical Deployment`

### 🚦 Lab 4 — Adaptive Traffic Light Controller
> Sensor-driven FSM with 5 states and real-time 7-segment countdown

A **5-state Moore FSM** managing highway and street traffic lights based on a car-presence sensor. The controller adapts timing dynamically — if no car is detected mid-cycle, it skips the street green phase entirely. Drives 6 LED outputs and a live 7-segment countdown, clocked at 48MHz with an on-chip 1Hz divider.

```verilog
localparam [2:0] s1=3'b000, s2=3'b001, s3=3'b010, s4=3'b011, s5=3'b100;
// s1: Highway green (waiting) → sensor triggers s2
// s2: Highway green + countdown (15s) → s3: Highway yellow (2s) → s4
// s4: Street green, sensor-adaptive (up to 5s) → s5: Street yellow → s1
```

`Moore FSM` `Clock Divider` `Multi-output Control` `Adaptive Logic`

### 🔬 Lab 5 — Real-Time Image Processing Pipeline
> Median denoising + Sobel edge detection — fully pipelined in hardware

The most complex lab. A **streaming pixel pipeline** that reads image data from ROM, passes it through a 3-pixel **median filter** (hardware bubble sort), then applies a **Sobel edge detector** using a 3-column kernel — all running simultaneously on the FPGA fabric, clocked and valid-signal synchronized.

```verilog
// Sobel kernel: |Gx| + |Gy| compared to threshold → binary edge map
always @(posedge clk_Sobel) begin
  Ax <= Sumx_1 - Sumx_3;                    // Gx accumulation
  Ay <= Sumy_3 + (Sumy_2 * 2) + Sumy_1;    // Gy accumulation
end
if (Ax_Abs + Ay_Abs > Threshold) Pixel_Sobel <= 255;
```

**Modules:** `Top.v` → `Resize.v` → `Median.v` → `Sobel.v` · Xilinx `CLOCKGEN` IP for domain crossing

`Pipelined Datapath` `Median Filter` `Sobel Edge Detection` `Clock Domain Crossing`

### 🔌 Lab 6 — FT245 USB FIFO Interface
> Talking to a host PC over USB — all in hardware logic

An **FT245 parallel FIFO controller** that receives a byte N from a host PC and streams back integers 1 to N — fully managed in RTL with no CPU involved. Handles tri-state bus arbitration, RXF/TXE handshake signals, and byte-level timing without any software stack.

```verilog
assign Data = Flag ? Counter : 8'bzzzzzzzz;  // tri-state bus control
// Read phase: RXF asserts → pulse RD → latch Data
// Write phase: while Counter ≤ Datareg → assert WR → output Counter
```

`USB/FIFO Protocol` `Tri-state Bus` `Handshake FSM` `Host-FPGA Comms`
